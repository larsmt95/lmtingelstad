Problemstilling

Ledelsen i Bike Store manglet oversikt over hva som faktisk driver salgsutviklingen. Omsetningen vokste, men det var uklart hvilke produkter og butikker som sto for veksten — og om veksten var bredt forankret eller konsentrert i et fåtall enheter.

ImportantHovedfunn

Salgsveksten er sterk (+42 % YoY), men tydelig konsentrert. En liten andel produkter driver en stor del av omsetningen, og én butikk står for en uforholdsmessig stor andel av salget. Veksten er primært volumdrevet — ikke prisdrevet.

Funn 01
Sterk vekst, men smal base
+42 % YoY er imponerende, men omsetningen er konsentrert i et begrenset antall produkter. Dette innebærer konsentrasjonsrisiko mot sortimentsendringer og leverandørproblemer.
+42% YoY
Funn 02
Én butikk dominerer
Én hovedbutikk står for en betydelig andel av totalomsetningen. Geografisk konsentrasjon begrenser vekstpotensialet og øker risikoen ved lokale markedsendringer.
Geo-risiko
Funn 03
Volumdrevet, ikke prisdrevet
Veksten forklares primært av økt salgsvolum. Det er rom for å vurdere prissetting og sortimentsoptimalisering for å forbedre marginen uten å ofre volum.

SQL — Datamodellering

Datastrukturen ble modellert som et stjerneskjema med dimensjonstabeller for produkt, butikk, selger og kunde, og én sentral faktatabell for salgstransaksjoner.

Dimensjonstabeller

-- Produktdimensjon med kategori og merke
CREATE OR REPLACE VIEW dim_products AS
SELECT
    p.product_id,
    p.product_name,
    c.category_name AS category,
    b.brand_name    AS brand,
    p.list_price
FROM products p
    JOIN brands     b USING (brand_id)
    JOIN categories c USING (category_id);

-- Butikkdimensjon
CREATE OR REPLACE VIEW dim_stores AS
SELECT
    store_id,
    store_name,
    city,
    state,
    CONCAT(city, ', ', state) AS location
FROM stores;

-- Selgerdimensjon med rolle-flagg
CREATE OR REPLACE VIEW dim_staff AS
SELECT
    staff_id,
    CONCAT(first_name, ' ', last_name) AS name,
    email,
    store_id,
    manager_id,
    CASE
        WHEN manager_id IS NULL THEN 'Manager'
        ELSE 'Staff'
    END AS role
FROM staffs
    JOIN stores USING (store_id);

Faktatabell

Faktatabellen samler alle salgshendelser på ordrenivå med både brutto- og nettoomsetning — noe som muliggjør analyse av rabatteffekter direkte i Power BI.

CREATE OR REPLACE VIEW fact_sales AS
SELECT
    o.order_id,
    o.order_date,
    o.store_id,
    o.staff_id,
    o.customer_id,
    oi.product_id,
    oi.quantity,
    oi.list_price,
    oi.discount,
    oi.quantity * oi.list_price                       AS gross_revenue,
    oi.quantity * oi.list_price * (1 - oi.discount)  AS net_revenue
FROM orders o
JOIN order_items oi USING (order_id);

Aggregerte analyse-views

For å forenkle Power BI-siden ble aggregerte views bygget direkte i SQL — slik at Power BI henter ferdige aggregater fremfor å beregne disse på importert rådata.

CREATE OR REPLACE VIEW vw_monthly_store_revenue AS
SELECT
    s.store_name,
    EXTRACT(YEAR  FROM order_date) AS year,
    EXTRACT(MONTH FROM order_date) AS month,
    SUM(gross_revenue)             AS gross_revenue,
    SUM(net_revenue)               AS net_revenue
FROM fact_sales f
    LEFT JOIN dim_stores s ON f.store_id = s.store_id
GROUP BY store_name, year, month
ORDER BY store_name, year, month;

Power BI — Dashboard

SQL-modellen brukes direkte som semantisk lag i Power BI — ingen transformasjon i Power Query utover datatyper.

NoteDatamodell i Power BI

Dashboardet er strukturert rundt tre analyser. Salgsutvikling over tid visualiserer månedlig omsetning og gjør det mulig å identifisere sesongmønstre og vekstperioder. Produkt- og kategorianalysen viser hvilke enheter som driver størstedelen av omsetningen og avdekker konsentrasjonsrisiko. Butikkanalysen sammenligner geografisk salgsprestasjon.

TipStrategisk tolkning

Høy konsentrasjon i produktportefølje og geografi er en risikofaktor som bør adresseres eksplisitt i sortiments- og ekspansjonsstrategi. Veksten er reell, men ikke bredt forankret.

Anbefalinger

📦
Reducer konsentrasjonsrisiko i sortimentet
Når en liten andel produkter driver en stor del av omsetningen, er avhengigheten av disse produktene høy. Aktiv bredding av sortimentet kan redusere risikoen ved leverandørproblemer eller etterspørselsfall.
🗺️
Vurder geografisk ekspansjon
Dominansen til én butikk indikerer uutnyttet potensial i andre geografier. En markedsanalyse av underrepresenterte regioner kan gi grunnlag for ekspansjonsbeslutninger.
💰
Optimaliser prissetting
Volumdrevet vekst er positivt, men prisoptimalisering på topprodukter kan forbedre margin uten å ofre volum. Rabattanalyse på produktnivå er et naturlig neste steg.
📊
Implementer løpende KPI-overvåkning
Dashboardet gir et godt grunnlag, men bør utvides med marginkPI-er på produktnivå og automatisk varsling ved avvik fra trendlinje.
